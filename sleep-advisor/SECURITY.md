# Security Policy

## Reporting a Vulnerability

The Lumina Sleep Advisor team takes security seriously. If you discover a security vulnerability in this project, please report it responsibly.

### How to Report

1. **Email:** Send a detailed report to [wagalad@users.noreply.github.com](mailto:wagalad@users.noreply.github.com)
2. **GitHub:** Open a [private security advisory](https://github.com/wagalad/Lumina/security/advisories/new) on this repository

### What to Include

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant screenshots or logs
- Suggested fix (if available)

### Response Timeline

| Stage | Timeframe |
|-------|-----------|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 1 week |
| Fix deployed | Within 2 weeks (critical) / 30 days (non-critical) |

### Scope

This policy applies to:
- The Lumina web application ([lumina-sleep.vercel.app](https://lumina-sleep.vercel.app) or equivalent deployment)
- This GitHub repository and its contents

### Out of Scope

- Third-party services linked in citations (e.g., sleepdata.org, kaggle.com)
- Issues in upstream libraries or CDNs
- Social engineering attacks

### Safe Harbor

We will not pursue legal action against security researchers who:
- Make a good-faith effort to follow this disclosure policy
- Avoid accessing or modifying other users' data
- Do not publicly disclose the vulnerability before a fix is deployed

---

Thank you for helping keep Lumina secure.
